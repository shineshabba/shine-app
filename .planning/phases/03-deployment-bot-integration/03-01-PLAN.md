**1. frontend/src/api/client.ts** — строка 4, заменить:
```
const API_BASE = 'https://api.shineee.space'
```
на:
```
const API_BASE = import.meta.env.VITE_API_URL ?? 'https://api.shineee.space'
```
Fallback на продакшн URL нужен чтобы локальная разработка без .env.local не ломала сборку.

**2. frontend/.env.example** — создать новый файл:
```
# API URL для Shine backend
# В Vercel dashboard: VITE_API_URL=https://api.shineee.space
# Для локальной разработки: VITE_API_URL=http://localhost:8000
VITE_API_URL=http://localhost:8000
```

**3. backend/main.py** — в CORSMiddleware добавить `https://app.shineee.space` в `allow_origins`. Текущий список:
```python
allow_origins=["http://localhost:5173", "http://localhost:3000"],
```
Заменить на:
```python
allow_origins=[
    "http://localhost:5173",
    "http://localhost:3000",
    "https://app.shineee.space",
],
```
Существующий allow_origin_regex для `*.vercel.app` оставить без изменений — он нужен для preview deployments.

Убедиться, что `allow_credentials=True` и `allow_headers=["X-Telegram-Init-Data", "Content-Type"]` остаются без изменений.
  </action>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Env vars → build | VITE_API_URL попадает в bundle во время Vite build — значение видно в JS |
| CORS → API | Список allowed origins в main.py — браузеры проверяют, curl игнорирует |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Information Disclosure | VITE_API_URL в bundle | accept | API_BASE — публичный URL, не секрет. Нет credentials в env. |
| T-03-02 | Spoofing | CORS allow_origins расширен | mitigate | Добавляем только `https://app.shineee.space` — конкретный домен, не wildcard. allow_origin_regex остаётся строгим (`^https://[a-zA-Z0-9][a-zA-Z0-9\-]*\.vercel\.app$`). |
| T-03-03 | Tampering | git push в public repo | accept | Репо публичный. Секреты хранятся в .env на VPS (не в репо) и в Vercel dashboard (не в коде). .env файлы в .gitignore. |
</threat_model>
