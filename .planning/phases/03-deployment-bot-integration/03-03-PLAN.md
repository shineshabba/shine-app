Типичный aiogram 3 паттерн для ReplyKeyboard с WebApp кнопкой:

```python
# Добавить импорт в начало файла (если не импортирован):
from aiogram.types import ReplyKeyboardMarkup, KeyboardButton, WebAppInfo

# В /start handler добавить клавиатуру:
webapp_keyboard = ReplyKeyboardMarkup(
    keyboard=[
        [KeyboardButton(
            text="🚀 Открыть приложение",
            web_app=WebAppInfo(url="https://app.shineee.space")
        )]
    ],
    resize_keyboard=True,
    one_time_keyboard=False,
)

await message.answer(
    "Добро пожаловать в Shine VPN!\n\n"
    "Нажмите кнопку ниже чтобы открыть приложение:",
    reply_markup=webapp_keyboard,
)
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| SSH root@VPS | Прямой доступ к серверу — любое изменение производственного кода |
| Telegram → WebApp | Telegram передаёт initData при открытии Mini App — доверяем Telegram HMAC |
| BotFather Menu Button | URL в BotFather — любой открывший бота увидит кнопку |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-09 | Tampering | SSH редактирование ~/vpn-bot/ | mitigate | Перед изменением создаётся резервная копия (*.bak). Изменения точечные — только добавление WebApp кнопки, существующая логика не трогается. |
| T-03-10 | Spoofing | WebAppInfo URL | accept | URL жёстко прописан как `https://app.shineee.space` — Vercel контролирует. Telegram проверяет что Mini App открывается с того же домена что зарегистрирован в BotFather. |
| T-03-11 | Denial of Service | Бот крашится после изменений | mitigate | Проверить `systemctl status` и `journalctl` после перезапуска. Если crash — немедленно `cp handlers/start.py.bak handlers/start.py && systemctl restart`. |
| T-03-12 | Elevation of Privilege | ReplyKeyboard кнопка всем юзерам | accept | По D-08 кнопка для всех — это осознанное решение. Mini App сам ограничивает доступ к данным на основе статуса подписки. |
</threat_model>
