**Шаг 1 — Установить Vercel CLI если нет:**
```bash
npm install -g vercel
```

**Шаг 2 — Залогиниться:**
```bash
vercel login
```
Выбрать метод аутентификации (GitHub или email).

**Шаг 3 — Задеплоить из корня монорепо:**
```bash
cd /path/to/shine-app  # корень репо
vercel --yes \
  --name shine-app \
  --build-env VITE_API_URL=https://api.shineee.space \
  --env VITE_API_URL=https://api.shineee.space
```

Если CLI спросит настройки — указать:
- **Root directory**: `frontend`
- **Build command**: `npm run build` (или `tsc -b && vite build`)
- **Output directory**: `dist`
- **Install command**: `npm install`

**Шаг 4 — Если нужен vercel.json** (для SPA routing, чтобы React Router работал):
Создать `frontend/vercel.json`:
```json
{
  "rewrites": [
    { "source": "/(.*)", "destination": "/index.html" }
  ]
}
```
Shine App — одностраничное приложение, роутинга нет (нет React Router), поэтому vercel.json не обязателен. Создать только если при переходе по прямому URL возникает 404.

**Шаг 5 — Проверить что VITE_API_URL установлен в Vercel:**
```bash
vercel env ls
```
Должна быть строка: `VITE_API_URL  https://api.shineee.space  Production, Preview, Development`

Если нет — добавить вручную:
```bash
vercel env add VITE_API_URL production
# Enter value: https://api.shineee.space
vercel env add VITE_API_URL preview
# Enter value: https://api.shineee.space
```

После добавления env vars — сделать redeploy:
```bash
vercel --prod
```

**Шаг 6 — Проверить деплой:**
```bash
vercel ls
```
Ожидаемый вывод: `shine-app  https://shine-app-*.vercel.app  ...`

**GitHub integration:** После первого деплоя через CLI, Vercel автоматически подключает GitHub repo для auto-deploy при push в main. Если не подключил — настроить в Vercel Dashboard → Project Settings → Git → Connect GitHub Repository → shineshabba/shine-app → Root Directory: frontend.

  <action>
**Цель:** CNAME `app.shineee.space → cname.vercel-dns.com` и SSL через Vercel.

**Шаг 1 — Добавить домен в Vercel project:**
```bash
vercel domains add app.shineee.space --scope shineshabba
```
Vercel покажет инструкцию для DNS и CNAME-значение (обычно `cname.vercel-dns.com`).

**Шаг 2 — Узнать текущего DNS провайдера для shineee.space:**
DNS для домена shineee.space настраивается у DNS-провайдера. Судя по Phase 1 (nginx + Let's Encrypt уже работает на VPS 205.172.56.163 для api.shineee.space), DNS управляется через Cloudflare или у регистратора.

Проверить:
```bash
whois shineee.space | grep -i "name server\|nserver"
```
или
```bash
dig NS shineee.space
```

**Шаг 3 — Добавить CNAME запись** (у DNS провайдера):
```
Type:  CNAME
Name:  app
Value: cname.vercel-dns.com
TTL:   300
```

Если Cloudflare — proxy status должен быть **DNS only** (серая тучка), НЕ Proxied (оранжевая). Cloudflare proxy + Vercel SSL конфликтуют.

**Шаг 4 — Дождаться propagation и подтвердить:**
```bash
# Проверить propagation (обычно 1-5 минут при TTL 300)
dig CNAME app.shineee.space
# Ожидаемый ответ: app.shineee.space. 300 IN CNAME cname.vercel-dns.com.

# Проверить что Vercel выдал SSL и домен активен:
vercel domains inspect app.shineee.space
# Ожидаемый статус: Valid Configuration
```

**Шаг 5 — Назначить домен на проект:**
```bash
vercel alias set shine-app.vercel.app app.shineee.space
# или через:
vercel --prod --target production
```

**Шаг 6 — Финальная проверка:**
```bash
curl -I https://app.shineee.space
```
Ожидаем: `HTTP/2 200` с заголовком `x-vercel-id`
