**Серия curl проверок для подтверждения работоспособности стека.**

**Проверка 1 — Backend health:**
```bash
curl -s https://api.shineee.space/api/health
```
Ожидаемый ответ: `{"status": "ok"}` или аналогичный JSON с HTTP 200.

**Проверка 2 — Backend /api/me без initData (должен вернуть 401 или 422):**
```bash
curl -s -o /dev/null -w "%{http_code}" -X GET https://api.shineee.space/api/me
```
Ожидаем: `401` или `422` — значит эндпоинт работает и отклоняет запросы без auth.

**Проверка 3 — Backend /api/me с пустым initData:**
```bash
curl -s -w "\nHTTP_CODE:%{http_code}" \
  -H "X-Telegram-Init-Data: " \
  https://api.shineee.space/api/me
```
Ожидаем: `401` с JSON `{"detail": "..."}`.

**Проверка 4 — Frontend доступен:**
```bash
curl -s -o /dev/null -w "%{http_code}" https://app.shineee.space
```
Ожидаем: `200`.

**Проверка 5 — Frontend содержит React root:**
```bash
curl -s https://app.shineee.space | grep -c 'id="root"\|<script'
```
Ожидаем: число > 0 (HTML содержит React mount point и JS).

**Проверка 6 — CORS headers от backend:**
```bash
curl -s -I \
  -H "Origin: https://app.shineee.space" \
  -H "Access-Control-Request-Method: GET" \
  -X OPTIONS \
  https://api.shineee.space/api/me
```
Ожидаем: заголовок `access-control-allow-origin: https://app.shineee.space` в ответе.

**Проверка 7 — SSL валидность:**
```bash
curl -v https://api.shineee.space/api/health 2>&1 | grep -E "issuer|subject|SSL|TLS|expire"
curl -v https://app.shineee.space 2>&1 | grep -E "issuer|subject|SSL|TLS|expire"
```
Ожидаем: оба сертификата от Let's Encrypt, не expired.

Если какая-либо проверка упала — зафиксировать и передать в SUMMARY с описанием проблемы.
