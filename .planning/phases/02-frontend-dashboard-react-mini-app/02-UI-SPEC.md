# Phase 2 — UI Design Contract

> Visual and interaction contract for Phase 2: Frontend Dashboard (React Mini App).
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.
> All decisions pre-populated from CONTEXT.md (D-01..D-20). No user questions asked (--auto mode).

## Design System

| Property | Value |
|----------|-------|
| Tool | none — Tailwind CSS 4 with Telegram CSS variables |
| Preset | not applicable |
| Component library | none — custom components via Tailwind CSS 4 utilities |
| Icon library | lucide-react (tree-shakeable, minimal bundle impact) |
| Font | System font stack inherited from Telegram — no custom font loaded |

**Rationale:** No `components.json` found in codebase. shadcn gate: project uses React 19 + Vite 6 + Tailwind CSS 4. Tailwind CSS 4 is incompatible with shadcn v2 (which requires Tailwind v3). Design tokens come from Telegram `themeParams` CSS variables, not from a design system preset. Source: CONTEXT.md D-10, D-11.

## Telegram Theme Integration

All colors derive from Telegram CSS variables injected by the Mini App runtime. The app MUST apply `--tg-theme-*` variables to every surface. No hardcoded hex colors on interactive or semantic elements.

| CSS Variable | Semantic Role |
|---|---|
| `--tg-theme-bg-color` | Card background, modal background |
| `--tg-theme-secondary-bg-color` | Page/screen background |
| `--tg-theme-text-color` | Primary text |
| `--tg-theme-hint-color` | Secondary text, labels, hints |
| `--tg-theme-button-color` | Accent — all interactive buttons |
| `--tg-theme-button-text-color` | Text on accent buttons |
| `--tg-theme-link-color` | Inline links |
| `--tg-theme-destructive-text-color` | Error messages, inactive subscription indicator |

Source: CONTEXT.md D-10, D-11; REQUIREMENTS.md FE-05, DASH-07.

## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-text gap, badge padding |
| sm | 8px | Inline element padding, button icon margin |
| md | 16px | Card inner padding, section gaps |
| lg | 24px | Block vertical spacing between Header/Subscription/Config/Support |
| xl | 32px | Top/bottom page padding |
| 2xl | 48px | QR modal inner padding |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Touch targets: minimum 44px height for all tappable elements (iOS HIG minimum, critical for mobile usability)
- Bottom sheet drag handle area: 20px height (4px handle + 16px padding each side)
- Config truncated link field: full-width, no padding restriction

## Typography

System font stack — no external fonts. All sizes in `rem` mapped to base 16px.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 16px (1rem) | 400 regular | 1.5 | Subscription details, config info, support text |
| Label | 14px (0.875rem) | 400 regular | 1.4 | Hints, device limit text, secondary labels |
| Heading | 20px (1.25rem) | 600 semibold | 1.2 | Section block titles (e.g. «Конфигурация», «Подписка») |
| Display | 28px (1.75rem) | 600 semibold | 1.2 | Not used in this phase (reserved for v2 AI screen) |

Only two font weights are used: **400 (regular)** and **600 (semibold)**.

## Color

Colors use Telegram CSS variables. The 60/30/10 model maps to Telegram's native palette:

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `var(--tg-theme-secondary-bg-color)` | Full-screen page background |
| Secondary (30%) | `var(--tg-theme-bg-color)` | Cards (Subscription block, Config block), bottom sheet, QR modal |
| Accent (10%) | `var(--tg-theme-button-color)` | See reserved-for list below |
| Success | `#34C759` (iOS green, override allowed in dark mode) | Active subscription badge and text |
| Destructive | `var(--tg-theme-destructive-text-color)` | Inactive subscription indicator only |

**Accent reserved for (explicit list):**
1. «Копировать» / «Скопировано ✓» button
2. «QR-код» button
3. «Оформить подписку» / «Продлить» CTA button
4. «Написать админу» button in bottom sheet

Accent is NOT used on: card borders, text links, section headings, device limit label, or header elements.

**Subscription status colors:**
- Active: `#34C759` text/badge for «Активна до DD.MM.YYYY» — source: CONTEXT.md D-12
- Inactive: `var(--tg-theme-destructive-text-color)` for «Не активна» — source: CONTEXT.md D-12

## Layout — Screen Structure

Single-screen SPA. No React Router. One DOM tree, conditional rendering for states.

```
┌─────────────────────────────┐
│ HEADER                      │  height: 56px, sticky top
│ [Имя юзера]      [иконка]   │
├─────────────────────────────┤
│ SUBSCRIPTION BLOCK          │  Card, rounded-2xl, bg-card
│ «Подписка»                  │
│ ✅ Активна до 20.06.2026    │
│ Лимит: 5 устройств          │
├─────────────────────────────┤
│ CONFIG BLOCK                │  Card (hidden if inactive)
│ «Конфигурация»              │
│ [vless://abc...def...]      │
│ [Копировать] [QR-код]       │
├─────────────────────────────┤
│ PAYMENT BUTTON              │  Visible only if inactive
│ [Оформить подписку]         │
├─────────────────────────────┤
│                             │  flex-grow spacer
│ [Поддержка ↗]               │  sticky bottom or last element
└─────────────────────────────┘
```

Source: CONTEXT.md D-01, D-05.

**Scroll behavior:** The screen scrolls vertically if content exceeds viewport. Support button is the last element in document flow (not fixed), scrolled to if needed. On short screens (320px), all blocks stack and scroll naturally.

## Component Inventory

### 1. Header

- Height: 56px
- Left: user `name` from `/api/me` — 20px semibold, `var(--tg-theme-text-color)`
- Right: Lucide icon `Settings2` or `Info` — 24px, `var(--tg-theme-hint-color)` (decorative, no action in v1 — Claude's discretion)
- Background: `var(--tg-theme-bg-color)` with 1px bottom border `var(--tg-theme-hint-color)` at 20% opacity
- Sticky: `position: sticky; top: 0; z-index: 10`

### 2. Subscription Block (Card)

- Background: `var(--tg-theme-bg-color)`
- Border radius: 16px (`rounded-2xl`)
- Padding: 16px
- Shadow: `0 2px 8px rgba(0,0,0,0.06)` — soft, telegram-native
- Title: «Подписка» — 20px/600
- Status line (active): «✅ Активна до DD.MM.YYYY» — 16px/400, color `#34C759`
- Status line (inactive): «❌ Не активна» — 16px/400, color `var(--tg-theme-destructive-text-color)`
- Device limit: «Лимит: 5 устройств» — 14px/400, `var(--tg-theme-hint-color)` — simple text, no progress bar

### 3. Config Block (Card) — shown only when `subscription_active: true`

- Same card styles as Subscription Block
- Title: «Конфигурация» — 20px/600
- VLESS display: monospace or code style text, truncated to ~40 chars + «…» — 14px/400, `var(--tg-theme-text-color)`, read-only, no editing
- Action row: two buttons side by side
  - «Копировать» — accent fill button, 44px min height
  - «QR-код» — accent fill button, 44px min height
- Copy feedback: button label changes to «Скопировано ✓» immediately on tap, reverts after 1500ms
- Buttons use `var(--tg-theme-button-color)` background, `var(--tg-theme-button-text-color)` text, border-radius 10px

### 4. Payment Button — shown only when `subscription_active: false`

- Full-width, 48px height, border-radius 12px
- Background: `var(--tg-theme-button-color)`, text: `var(--tg-theme-button-text-color)`
- Label: «Оформить подписку» (new user) or «Продлить подписку» (lapsed) — determined by whether `subscription_end` field is populated
- Action: `tg.openTelegramLink('https://t.me/tribute/app?startapp=sVHH')`
- Font: 16px/600

### 5. Support Button

- Positioned at bottom of content flow
- Style: ghost/outline — `var(--tg-theme-link-color)` text, no fill, no border
- Layout: Lucide `HeadphonesIcon` 18px + «Поддержка» text — 16px/400
- Min height: 44px
- Action: opens bottom sheet (see §6)

### 6. Bottom Sheet — Support

- Slides up from bottom with CSS `transform: translateY(0)` transition, 300ms ease-out
- Backdrop: `rgba(0,0,0,0.4)` full-screen, tap outside closes sheet
- Sheet background: `var(--tg-theme-bg-color)`, top border-radius 20px
- Drag handle: 4px × 36px, `var(--tg-theme-hint-color)` at 40% opacity, centered at top
- Max-height: 80vh, inner content scrollable
- Close: tap backdrop, or tap «×» icon top-right (24px)

**Accordion sections — 4 items:**

| Platform | Icon (Lucide) | Title |
|---|---|---|
| iOS | `Smartphone` | iPhone / iPad |
| Android | `Tablet` | Android |
| Windows | `Monitor` | Windows |
| Mac | `Laptop` | macOS |

Each accordion item:
- Header: 16px/600, 44px min height, chevron right/down indicator
- Expanded content: 14px/400, `var(--tg-theme-hint-color)`, padding 12px 16px
- Default state: all collapsed

**Instruction copy (Claude's discretion defaults):**
- iOS: «Скопируй конфигурацию → открой Streisand или Shadowrocket → нажми + → вставь ссылку»
- Android: «Скопируй конфигурацию → открой v2rayNG или Hiddify → нажми + → вставь ссылку»
- Windows: «Скопируй конфигурацию → открой Hiddify или v2rayN → добавь сервер из буфера»
- Mac: «Скопируй конфигурацию → открой Hiddify или FoXray → добавь сервер из буфера»

**Admin button** (below all accordion items):
- Label: «Написать администратору»
- Style: full-width, 48px, accent fill (same as Payment Button)
- Action: `tg.openTelegramLink('https://t.me/shineshabba')`

### 7. QR Modal — Full-Screen Overlay

- Full-screen overlay: `position: fixed; inset: 0; z-index: 50`
- Background: `var(--tg-theme-bg-color)` (white in light, dark in dark theme)
- QR code: centered, size = `min(80vw, 80vh)` — maximum visible without scrolling
- Library: `qrcode.react` (lightweight, ~15KB gzip, generates SVG)
- Close affordances:
  - Tap anywhere outside QR to close (full overlay is tappable)
  - «×» button top-right, 44px touch target, `var(--tg-theme-hint-color)`
- No title, no border — just QR on clean background
- Entry: fade-in 150ms opacity 0→1
- Exit: fade-out 150ms opacity 1→0

### 8. Loading State

- Shown while both `/api/me` and `/api/config` are in flight
- Full-screen centered: Telegram-style animated spinner (CSS animation, no library)
- Spinner: 32px, `var(--tg-theme-button-color)`, 3-dot pulse or arc spinner
- No skeleton screens (bundle budget — spinner is sufficient for <2s load)

### 9. Error State

- Shown when API call fails with network error or 5xx
- Layout: centered, flex column
- Icon: Lucide `WifiOff` 48px, `var(--tg-theme-hint-color)`
- Heading: «Не удалось загрузить данные» — 20px/600
- Body: «Проверьте соединение и попробуйте снова» — 16px/400, `var(--tg-theme-hint-color)`
- Button: «Повторить» — accent fill, 48px height, calls API again

### 10. Unregistered User Screen (404 from `/api/me`)

- Layout: centered flex column, full height
- Icon: Lucide `UserX` 48px, `var(--tg-theme-hint-color)`
- Heading: «Аккаунт не найден» — 20px/600
- Body: «Пройдите /start в боте @shine_connect_bot» — 16px/400, `var(--tg-theme-hint-color)`
- No buttons (user must go to bot)

## API Error Handling Map

| HTTP Status | Source | UI Behavior |
|-------------|--------|-------------|
| 404 | `/api/me` | Show Unregistered User screen (§10) |
| 403 | `/api/config` | Hide Config Block, show Payment Button — `subscription_active` already false |
| 401 | Any endpoint | Show error screen with «Не удалось авторизоваться» message |
| Network error / 5xx | Any endpoint | Show Error State screen (§9) with «Повторить» button |

## Performance Budget

| Metric | Constraint | Approach |
|--------|------------|----------|
| Bundle gzipped | < 300KB | No heavy UI libraries; lucide-react tree-shaken; qrcode.react ~15KB; @telegram-apps/sdk-react ~20KB |
| Open time 4G | < 2s | Vite code splitting; zero Google Fonts; minimal animation (CSS only); no icon fonts |
| Animation | CSS only | No Framer Motion, no GSAP — `transition` utilities via Tailwind only |
| Images | None | Zero images. QR generated in-browser. Icons via lucide-react SVG components |

## Responsive Constraints

- Minimum viewport: 320px (iPhone SE / old Androids)
- Maximum tested: 430px (iPhone 15 Pro Max)
- Telegram Mini App typically opens as full-screen modal — assume 100dvh available
- All touch targets: minimum 44px × 44px
- No hover states — mobile-only interaction model
- Cards: `mx-16px` horizontal margin, full width minus margins

---

## Accessibility

- All interactive elements have `aria-label` in Russian
- Color contrast: Telegram theme variables maintain WCAG AA by Telegram's design
- `✅` / `❌` emoji must be accompanied by text (they are — full status string includes text)
- Bottom sheet: focus trap while open; `Escape` key closes (desktop Telegram)
- QR modal: `role="dialog"`, `aria-label="QR-код конфигурации"`, focus trap
