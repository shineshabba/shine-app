<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Telegram → Mini App | initData передаётся от Telegram клиента в webApp.initData — подлинность гарантируется HMAC на backend |
| Mini App → API | Frontend отправляет initData как заголовок X-Telegram-Init-Data — backend не доверяет без HMAC проверки |
| API Response → DOM | VLESS URL и имя пользователя из API рендерятся в DOM — риск XSS при неправильном рендеринге |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Spoofing | X-Telegram-Init-Data header | mitigate | Backend выполняет HMAC-SHA256 проверку (реализовано в Phase 1); frontend никогда не конструирует initData вручную — использует только `window.Telegram.WebApp.initData` |
| T-02-02 | Information Disclosure | VLESS URL в DOM | mitigate | React использует textContent (не innerHTML) для рендеринга строк — XSS невозможен при использовании JSX. Запрещено использовать dangerouslySetInnerHTML для данных из API |
| T-02-03 | Information Disclosure | VLESS URL в буфере обмена | accept | Пользователь намеренно копирует конфиг — это ожидаемое поведение. Буфер обмена — не threat boundary для данного use case |
| T-02-04 | Tampering | localStorage / sessionStorage | accept | Приложение не хранит initData и не кэширует пользовательские данные в localStorage. Каждый запуск = свежий fetch |
| T-02-05 | Elevation of Privilege | user_id в API-запросах | mitigate | Frontend НИКОГДА не передаёт tg_user_id в query params или body — только initData в заголовке. Backend извлекает user_id исключительно из валидированного initData (ASVS L1: V2.1) |
| T-02-06 | Denial of Service | Многократный fetch при ошибке | mitigate | Кнопка «Повторить» требует явного действия пользователя — нет автоматического ретрая. Не используются полинг или WebSocket в этой фазе |
| T-02-07 | Information Disclosure | initData в URL / логах | mitigate | initData передаётся только в HTTP-заголовке X-Telegram-Init-Data, никогда в URL query string или теле запроса |
</threat_model>
