<task type="checkpoint:human-verify" gate="blocking">
  <name>Task 2: Human checkpoint — визуальная проверка всех состояний Mini App</name>
  <what-built>
Полный React Mini App дашборд с:
- Header с именем пользователя
- Блок подписки (статус активна/не активна, лимит устройств)
- Блок конфигурации (VLESS truncated, «Копировать», «QR-код»)
- Кнопка «Оформить/Продлить подписку» (для неактивных)
- Кнопка «Поддержка» → bottom sheet с accordion
- QR Modal full-screen
- Loading, Error, Unregistered экраны
- Telegram theme (light/dark)
  </what-built>
  <how-to-verify>
**Шаг 1: Запустить dev-сервер**
```bash
cd frontend && npm run dev
# Открыть http://localhost:3000 в браузере
```

**Шаг 2: Проверить Loading state**
- Открыть страницу — должен появиться спиннер пока грузятся данные

**Шаг 3: Проверить UI (используется fallback theme — светлая тема)**
- Header: должен показывать имя (или «Загрузка...» если нет Telegram)
- Блок «Подписка»: должен быть виден с заголовком
- Цвета: фон страницы светло-серый, карточки белые
- Кнопки: синие (--tg-theme-button-color fallback = #2481cc)

**Шаг 4: Проверить интерактивность (вне Telegram — API вернёт 422/401)**
- Error state должен появиться — «Не удалось загрузить данные» + кнопка «Повторить»
- Нажать «Повторить» — должен снова показаться спиннер

**Шаг 5: Проверить QR Modal (если удастся смоделировать ready state)**
- Если есть способ передать mock данные — нажать «QR-код»
- Должен открыться full-screen overlay
- Нажать × или вне QR — должен закрыться

**Шаг 6: Проверить SupportSheet**
- Нажать «Поддержка» → bottom sheet должен выехать снизу
- Нажать «iPhone / iPad» → должен раскрыться accordion
- Нажать снова → должен закрыться
- Нажать за пределами sheet → должен закрыться

**Шаг 7: Проверить build**
```bash
cd frontend && npm run build
# Проверить размер в dist/assets/ — суммарно ≤ 300KB
```

**Ожидаемое поведение:**
- Все тексты на русском
- Нет слов VPN/прокси
- Спиннер — CSS анимация без библиотек
- Переходы плавные (bottom sheet 300ms slide-up)
  </how-to-verify>
