  ПРОВЕРКА 3: SSL сертификат валиден
  ```bash
  curl -sI https://api.shineee.space/api/health 2>&1 | head -5
  # PASS если: нет SSL ошибок, HTTP/2 200
  echo | openssl s_client -connect api.shineee.space:443 2>/dev/null | openssl x509 -noout -dates -issuer
  # PASS если: issuer содержит "Let's Encrypt"
  ```

  ПРОВЕРКА 4: Отклонение подделанного initData (AUTH-01)
  ```bash
  curl -s -H "X-Telegram-Init-Data: auth_date=1234567890&user=%7B%22id%22%3A123%7D&hash=fakehash000" https://api.shineee.space/api/me
  # PASS если: HTTP 401, {"detail":"Invalid initData signature"}
  curl -s -o /dev/null -w "%{http_code}" -H "X-Telegram-Init-Data: auth_date=1234567890&user=%7B%22id%22%3A123%7D&hash=fakehash000" https://api.shineee.space/api/me
  # PASS если: 401
  ```

  ПРОВЕРКА 5: Отклонение истёкшего initData (AUTH-05)
  Сгенерировать данные с auth_date 48 часов назад:
  ```bash
  python3 -c "
  import hashlib, hmac, json, os, urllib.parse, time
  bot_token = open('/root/shine-app/backend/.env').read()
  # Извлечь BOT_TOKEN
  bot_token = [l.split('=',1)[1].strip() for l in open('/root/shine-app/backend/.env') if l.startswith('BOT_TOKEN=')][0]
  auth_date = int(time.time()) - 172800  # 48 hours ago
  user = json.dumps({'id': 123456789, 'first_name': 'Test'})
  params = {'auth_date': str(auth_date), 'user': user}
  dcs = '\n'.join(f'{k}={v}' for k, v in sorted(params.items()))
  sk = hmac.new(b'WebAppData', bot_token.encode(), hashlib.sha256).digest()
  params['hash'] = hmac.new(sk, dcs.encode(), hashlib.sha256).hexdigest()
  print(urllib.parse.urlencode(params))
  "
  ```
  Использовать полученную строку:
  ```bash
  EXPIRED_INIT_DATA="<вставить результат python3 команды>"
  curl -s -o /dev/null -w "%{http_code}" -H "X-Telegram-Init-Data: $EXPIRED_INIT_DATA" https://api.shineee.space/api/me
  # PASS если: 401
  ```
