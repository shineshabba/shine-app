  ```
  venv/
  __pycache__/
  *.pyc
  .pytest_cache/
  *.egg-info/
  dist/
  build/
  ```

  3. Создать ~/shine-app/.gitignore (корневой):
  ```
  backend/.env
  backend/venv/
  ```

  4. Создать ~/shine-app/backend/.env:
  Скопировать реальные значения из ~/vpn-bot/.env (если существует):
  ```bash
  cat ~/vpn-bot/.env 2>/dev/null || echo "vpn-bot .env not found"
  ```

  Создать .env с реальными значениями (заполнить из vpn-bot/.env):
  ```
  # Telegram (из BotFather / vpn-bot/.env)
  BOT_TOKEN=<скопировать из ~/vpn-bot/.env>

  # SQLite database (read-only, per D-04)
  DB_PATH=/root/vpn-bot/users.db

  # x-ui panel (из vpn-bot/.env или x-ui admin)
  XUI_BASE_URL=http://127.0.0.1:21008
  XUI_PASSWORD=<скопировать из ~/vpn-bot/.env>
  XUI_INBOUND_ID=2

  # Добавить все остальные переменные из 01-01-SUMMARY.md
  ```

  ВАЖНО: НЕ оставлять placeholder-значения типа <скопировать> — заполнить реальными значениями.
  Если vpn-bot/.env не читается — запросить значения у пользователя через checkpoint.

  5. Создать /etc/systemd/system/shine-app-backend.service (per RESEARCH.md Pattern 3):
  ```ini
  [Unit]
  Description=Shine App Backend (FastAPI)
  After=network.target
  # Независимо от бота — не зависим от vpn-bot.service если он есть

  [Service]
  User=root
  WorkingDirectory=/root/shine-app/backend
  EnvironmentFile=/root/shine-app/backend/.env
  ExecStart=/root/shine-app/backend/venv/bin/uvicorn main:app --host 127.0.0.1 --port 8000 --workers 1
  Restart=always
  RestartSec=5
  StandardOutput=journal
  StandardError=journal
  SyslogIdentifier=shine-app-backend

  [Install]
  WantedBy=multi-user.target
  ```

  ВАЖНО: --host 127.0.0.1 (не 0.0.0.0) — только localhost, nginx проксирует снаружи.
  ВАЖНО: EnvironmentFile с абсолютным путём /root/shine-app/backend/.env (per RESEARCH.md Pitfall 6).
  ВАЖНО: Если VPS-пользователь не root — заменить User= и все пути /root/ соответственно.

  6. Включить и запустить сервис:
  ```bash
  systemctl daemon-reload
  systemctl enable shine-app-backend
  systemctl start shine-app-backend
  sleep 3
  systemctl status shine-app-backend
  ```

  7. Проверить что uvicorn отвечает:
  ```bash
  curl -s http://127.0.0.1:8000/api/health
  # ожидается: {"status":"ok"}
  ```

  8. Проверить логи на ошибки:
  ```bash
  journalctl -u shine-app-backend -n 20 --no-pager
  ```
  Если есть ошибки ImportError или KeyError (отсутствуют env vars) — исправить .env и перезапустить:
  ```bash
  systemctl restart shine-app-backend
  ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| systemd → .env файл | systemd читает секреты из EnvironmentFile; файл должен быть с ограниченными правами |
| uvicorn → network | Bind только на 127.0.0.1 — прямой внешний доступ невозможен без nginx |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Information Disclosure | .env файл права доступа | mitigate | chmod 600 ~/shine-app/backend/.env — читать может только root/owner сервиса |
| T-03-02 | Information Disclosure | .env в git | mitigate | .gitignore содержит .env; .env.example содержит только имена переменных без значений |
| T-03-03 | Denial of Service | uvicorn на 0.0.0.0 | mitigate | --host 127.0.0.1 в ExecStart; внешний трафик только через nginx с rate limiting |
| T-03-04 | Elevation of Privilege | сервис запущен как root | accept | VPS используется одним пользователем (19 users сервис); при масштабировании создать выделенного user |
| T-03-05 | Tampering | автоматический рестарт при краше | accept | Restart=always — при атаке краш-лупом сервис будет перезапускаться; для 19 users приемлемо |
</threat_model>
