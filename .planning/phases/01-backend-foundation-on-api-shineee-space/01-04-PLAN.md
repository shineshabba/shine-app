  <action>
  ВЫПОЛНЯЕТСЯ НА VPS через SSH.

  1. Проверить существующие сертификаты:
  ```bash
  ls /etc/letsencrypt/live/
  # Если есть *.shineee.space (wildcard) — можно использовать его
  # Если только shineee.space — нужен новый cert для api.shineee.space
  ```

  2. Создать nginx vhost /etc/nginx/sites-available/api.shineee.space (per RESEARCH.md Pattern 4):

  ВАРИАНТ A — если нет wildcard cert (нужен новый):
  ```nginx
  server {
      listen 80;
      server_name api.shineee.space;
      return 301 https://$host$request_uri;
  }

  server {
      listen 443 ssl;
      server_name api.shineee.space;

      ssl_certificate /etc/letsencrypt/live/api.shineee.space/fullchain.pem;
      ssl_certificate_key /etc/letsencrypt/live/api.shineee.space/privkey.pem;

      # Security headers
      add_header X-Frame-Options "DENY" always;
      add_header X-Content-Type-Options "nosniff" always;

      location / {
          proxy_pass http://127.0.0.1:8000;
          proxy_set_header Host $host;
          proxy_set_header X-Real-IP $remote_addr;
          proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
          proxy_set_header X-Forwarded-Proto $scheme;
          proxy_read_timeout 30s;
          proxy_connect_timeout 5s;
      }
  }
  ```

  ВАРИАНТ B — если есть wildcard cert *.shineee.space:
  Заменить пути сертификата на существующий wildcard cert путь.
  Смотреть в /etc/letsencrypt/live/ для точного пути.

  3. Включить vhost:
  ```bash
  ln -sf /etc/nginx/sites-available/api.shineee.space /etc/nginx/sites-enabled/api.shineee.space
  nginx -t
  # Expected: syntax is ok, test is successful
  ```

  4. Получить SSL сертификат (если нет wildcard):
  ```bash
  # Убедиться что certbot установлен
  certbot --version || apt install -y certbot python3-certbot-nginx

  # Получить cert для api.shineee.space
  certbot --nginx -d api.shineee.space --non-interactive --agree-tos --email shabaldas1@gmail.com
  ```
  Certbot автоматически обновит nginx конфиг с путями сертификата.

  5. Перезагрузить nginx:
  ```bash
  systemctl reload nginx
  nginx -t && systemctl reload nginx
  ```

  6. Финальная проверка:
  ```bash
  curl -v https://api.shineee.space/api/health 2>&1 | grep -E "(HTTP|status|SSL|certificate)"
  # Expected: HTTP/2 200, {"status":"ok"}
  ```

  Если certbot не может получить cert (DNS не распространился) — подождать и повторить.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Internet → nginx | Внешние HTTP/HTTPS запросы; nginx терминирует SSL |
| nginx → uvicorn | Внутренний проксированный трафик localhost:8000 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Information Disclosure | nginx без security headers | mitigate | X-Frame-Options: DENY, X-Content-Type-Options: nosniff в vhost конфиге |
| T-04-02 | Spoofing | HTTP запрос без TLS | mitigate | HTTP → HTTPS redirect (301) в nginx vhost |
| T-04-03 | Tampering | устаревший SSL cert | mitigate | certbot auto-renewal через cron (certbot устанавливает автоматически при install) |
| T-04-04 | Denial of Service | proxy_read_timeout | mitigate | proxy_read_timeout 30s — прерывает зависшие upstream соединения |
| T-04-05 | Information Disclosure | nginx version в заголовках | accept | server_tokens off не критично для этого сервиса; добавить в v2 hardening |
</threat_model>
